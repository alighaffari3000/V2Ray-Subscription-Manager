<h1 align="center">V2RayDAR</h1>

<p align="center">
  <em>Обнаружение и разведка V2Ray — произносится как <code>v2ray</code> + <code>radar</code>.</em>
</p>

<p align="center">
  Быстрый CLI/TUI на Rust, который загружает источники подписок V2Ray / Clash / Mihomo, проверяет конфигурации через вашу реальную сеть с помощью <code>sing-box</code>, ранжирует рабочие конфигурации и повторно публикует лучшие на локальном URL подписки для клиентов v2rayN / v2rayNG / sing-box / Clash Verge / Mihomo.
</p>

## 🤔 Почему V2RayDAR

- Параллельная загрузка подписок из неограниченного количества источников.
- Поддержка сырых, base64, JSON и YAML форматов — а также ссылок `vmess`, `vless`, `trojan`, `ss`, `ssr`, `hysteria2`, `hy2`, `tuic`.
- **Поддержка конфигураций Clash/Mihomo YAML** — добавьте URL подписки Mihomo, и V2RayDAR автоматически извлечёт все записи прокси.
- **Двунаправленное преобразование форматов** — конвертирует между ссылками V2Ray и записями прокси Clash/Mihomo YAML.
- Проверка каждого кандидата через вашу реальную сеть с `sing-box` (реально загружает тестовый URL через прокси).
- **Двухформатный вывод** — предоставляет рабочие конфигурации как ссылки V2Ray (`/subscription`) **и** полные конфиги Mihomo YAML (`/mihomo.yaml`).
- Повторная публикация лучших рабочих конфигураций на локальном URL для любого совместимого клиента.
- Выживание в ограниченных сетях через ранее проверенные конфигурации в базе данных, мостовую конфигурацию сети или `emergency_config`.
- Необязательный общий доступ по LAN с необязательной защитой токеном для использования одного источника с телефона.

## 📦 Быстрая установка

Скопируйте команду для вашей ОС в терминал. Установочный скрипт определяет платформу, загружает последний релиз с `sing-box` и настраивает всё. Портативный режим устанавливает в `Desktop/V2RayDAR` (если есть папка рабочего стола), иначе в `~/V2RayDAR`. Пользовательский режим устанавливает бинарник в `~/.local/bin`.

**Портативный режим** (рекомендуется) — все файлы в одной папке, запуск с `--portable`:
```bash
# Linux
curl -fsSL https://raw.githubusercontent.com/411A/V2RayDAR/main/install.sh | sh

# macOS
curl -fsSL https://raw.githubusercontent.com/411A/V2RayDAR/main/install.sh | sh

# Windows
irm https://raw.githubusercontent.com/411A/V2RayDAR/main/install.ps1 | iex
```

**Пользовательская установка** — бинарник в `~/.local/bin`, данные в домашней директории:
```bash
# Linux / macOS
curl -fsSL https://raw.githubusercontent.com/411A/V2RayDAR/main/install.sh | sh -s -- --user

# Windows
irm https://raw.githubusercontent.com/411A/V2RayDAR/main/install.ps1 | iex
# Затем выберите вариант 2 при запросе
```

**Android / Termux:**
```bash
# Установите sing-box, затем запустите установочный скрипт
pkg update -y && pkg install -y curl tar sing-box=1.13.13
curl -fsSL https://raw.githubusercontent.com/411A/V2RayDAR/main/install.sh | sh
# Всегда используйте --no-tui в Termux (ввод мыши не работает в терминалах Termux)
cd V2RayDAR && ./v2raydar --no-tui
```

**Ручная загрузка** — скачайте архив для вашей ОС из [Releases](https://github.com/411A/V2RayDAR/releases/latest) и запустите с `--portable`.

Установочный скрипт проверяет SHA-256 чек-суммы, определяет установленные копии и предлагает обновление (сохраняя `configs.yaml`, `data.db` и `v2raydar_data/`), по умолчанию не требует sudo.

## 🔰 Быстрый старт

После установки запустите `v2raydar` (в Windows — `v2raydar.exe`). При первом запуске создаётся `configs.yaml` с предварительно выбранными источниками подписок.

1. **Дождитесь заполнения.** Приложение загружает источники параллельно, проверяет каждую конфигурацию через вашу сеть и ранжирует рабочие. Эндпоинт доступен с самого начала — клиент может сразу подключиться.
2. **Укажите клиенту** URL подписки:

| Клиент | Эндпоинт |
| --- | --- |
| v2rayN / v2rayNG | `http://127.0.0.1:27141/subscription` (base64) |
| sing-box | `http://127.0.0.1:27141/subscription.txt` (текстовый) |
| Clash Verge / Mihomo | `http://127.0.0.1:27141/mihomo.yaml` |

3. **Управление TUI:**

| Клавиша | Действие |
| --- | --- |
| `↑` / `↓` или `j` / `k` | Навигация |
| `Enter` | Выбор / переключение / подтверждение |
| `Esc` / `Ctrl+H` | Назад |
| `Space` | Переключение подписки |
| `e` | Редактировать выбранную подписку |
| `q` | Выход |
| `:` | Режим команд — `:q` выход, `:w` сохранить, `:a` добавить, `:d` удалить, `:n` переименовать, `:u` URL, `:p` приоритет |

4. **Изменение настроек** через главное меню TUI (Configurations) или редактирование `configs.yaml` напрямую — изменения вступят в силу при следующем обновлении. Ключевые настройки: `top_n`, `refresh_seconds`, `sharing.enabled`, `probe.mode`.
5. **Выход** по `q` или `:q`. После выхода эндпоинт останавливается.

### Режимы запуска

```bash
v2raydar                # TUI + локальный эндпоинт подписки
v2raydar --no-tui       # без интерфейса — только эндпоинт и логи
v2raydar --once         # одно обновление, вывод результатов, выход
v2raydar --portable     # данные рядом с исполняемым файлом
v2raydar --uninstall    # удаление данных приложения и правил файрвола
```

Windows-пользователи заменяют `v2raydar` на `v2raydar.exe`. В macOS откройте вложенный `.app` один раз — Gatekeeper запомнит его.

## ⚙️ Конфигурация по умолчанию

<details>
  <summary>👣 <strong>configs.yaml</strong> — таблица всех ключей, значений по умолчанию и их назначения. Подробности в <a href="guide.md">руководстве разработчика</a>.</summary>

| Ключ | По умолчанию | Назначение |
| --- | --- | --- |
| `bind` | `127.0.0.1:27141` | Локальный HTTP-адрес для `/subscription`, `/subscription.txt`, `/results` и `/health`. |
| `top_n` | `10` | Количество рабочих конфигураций для публикации клиентам. |
| `refresh_seconds` | `300` | Интервал автообновления (секунды); `0` отключает таймер. |
| `encoded_subscription` | `true` | `/subscription` возвращает base64 (совместимо с v2rayN / v2rayNG). |
| `prioritize_stability` | `true` | Повторно проверяет сохранённый Top-N предыдущего запуска и сохраняет их впереди, даже если новые конфигурации имеют меньшую задержку. При `false` предпочтение отдаётся любой рабочей конфигурации с низкой задержкой. |
| `return_configs_asap` | `false` | При `true` рабочие конфигурации публикуются сразу после обнаружения (макс. `top_n`); ранние конфигурации могут не иметь наилучшей задержки или стабильности. |
| `scan_all_configs` | `false` | При `true` проверяет все загруженные конфигурации, а не останавливается при достаточном количестве. |
| `fetch_timeout_ms` | `30000` | Таймаут загрузки каждого источника. |
| `fetch_concurrency` | `8` | Количество источников загружаемых параллельно. |
| `max_subscription_bytes` | `33554432` | Максимальный размер каждого источника (32 МиБ). |
| `use_cache_only` | `false` | Пропускает загрузку и берёт ранее проверенные конфигурации из базы данных — полезно в ограниченных сетях. |
| `emergency_config` | `null` | Необязательная рабочая ссылка для использования через `sing-box` в качестве моста при неудачной загрузке HTTP-подписок. |
| `clean_offlines_after_days` | `7` | Дни, после которых недоступные конфигурации удаляются из базы данных. |
| `sharing.enabled` | `false` | Разрешает клиентам LAN доступ к эндпоинтам. |
| `sharing.require_token` | `false` | Запросы LAN требуют `?token=...`. |
| `sharing.token` | `null` | Пусто = отключён, `true` = авто-генерация, строка = указанное значение. |
| `proxy.enabled` | `false` | Запускает постоянный процесс SOCKS5/HTTP прокси через `sing-box`. |
| `proxy.port` | `27910` | Порт смешанного SOCKS5/HTTP прокси. |
| `proxy.discoverable` | `false` | Привязывается к `0.0.0.0` и добавляет правило файрвола для доступа по LAN. |
| `proxy.health_check_url` | `https://www.gstatic.com/generate_204` | URL для проверки работоспособности прокси. |
| `proxy.health_check_interval_seconds` | `60` | Секунды между проверками работоспособности. Автопереключение при сбое. |
| `probe.mode` | `active` | `active` использует `sing-box`; `tcp` только для диагностики. |
| `probe.sing_box_path` | `null` | Необязательный путь к `sing-box`. Для десктопных сборок `_with_singbox` или пакета Termux оставьте `null`. |
| `probe.connect_timeout_ms` | `5000` | Таймаут TCP-подключения в диагностическом режиме. |
| `probe.active_timeout_ms` | `30000` | Таймаут HTTP-теста в активном режиме. |
| `probe.startup_timeout_ms` | `5000` | Время ожидания запуска временного прокси. |
| `probe.concurrency` | `16` | Базовая并发 активной проверки. |
| `probe.batch_size` | `20` | Начальный размер партии активной проверки. |
| `probe.process_concurrency` | `null` | Количество одновременных процессов `sing-box`; при пустом значении масштабируется автоматически. |
| `probe.test_url` | `https://www.gstatic.com/generate_204` | Тестовый URL, загружаемый через каждого кандидата. |
| `probe.accepted_statuses` | `[204, 200]` | HTTP-коды, считающиеся успешными. |
| `probe.download_url` | `null` | Необязательная цель теста пропускной способности. |
| `probe.download_bytes_limit` | `1048576` | Максимум байт за тест скорости. |
| `geoip_db_path` | `null` | Необязательный путь к файлу `GeoLite2-Country.mmdb`. При `null` используется встроенная база для определения страны. |
| `subscriptions` | _(предвыбранные источники)_ | Список источников `{ name, url, enabled, priority }`. Добавьте свои для лучшего покрытия. |

</details>

## 🌐 Заметки о ограниченных сетях

- В ограниченных сетях ранее проверенные конфигурации хранятся в базе данных и доступны через `use_cache_only: true`.
- По умолчанию, если некоторые HTTP-URL подписок недоступны, но есть рабочая конфигурация, приложение использует её для повторной попытки загрузки. Если рабочих конфигураций нет, но у вас есть одна рабочая, добавьте её в `emergency_config` в `configs.yaml`.

## 📡 Подключение клиентов к V2RayDAR

- **v2rayN (тот же ПК)** — оставьте `bind: 127.0.0.1:27141` и добавьте `http://127.0.0.1:27141/subscription` как URL подписки.
- **v2rayNG / телефон в той же Wi-Fi** — привяжитесь к LAN-IP ПК (например, `192.168.1.23:27141`), включите `sharing.enabled`, затем используйте `http://192.168.1.23:27141/subscription` на телефоне. Сначала проверьте `/health` с телефона.

Подробное руководство по настройке клиентов, защите токеном и файрволу для каждой ОС см. в [руководстве разработчика](guide.md).

### 📱 Постоянный прокси для трафика приложений

V2RayDAR может запускать постоянный SOCKS5/HTTP прокси рядом с эндпоинтом подписки. Любое приложение на системе — Telegram, браузеры, curl, Python — может маршрутизировать трафик через него без отдельного VPN-клиента.

**Включите в `configs.yaml`:**
```yaml
proxy:
  enabled: true
  port: 27910
  discoverable: false   # true = доступ по LAN + правило файрвола
```

**Локальное использование (на устройстве с V2RayDAR):**
```bash
curl --socks5 127.0.0.1:27910 https://api.ipify.org
```

**Использование по LAN (телефон в той же Wi-Fi):**
1. Установите `proxy.discoverable: true` — V2RayDAR добавит правило файрвола и привяжется к `0.0.0.0`.
2. Найдите LAN IP устройства с V2RayDAR в панели TUI в разделе **Network** (или выполните `ipconfig` / `ip addr`). Например `192.168.1.2`.
3. **Telegram:** замените `YOUR_LAN_IP` на ваш реальный LAN IP и откройте этот URL на телефоне:

   ```
   https://t.me/socks?server=YOUR_LAN_IP&port=27910
   ```

   Например, если ваш LAN IP — `192.168.1.2`:
   ```
   https://t.me/socks?server=192.168.1.2&port=27910
   ```

   Или вручную: Telegram → Настройки → Данные и память → Настройки прокси → Добавить прокси:
   - Тип: **SOCKS5** или **HTTP**
   - Хост: `YOUR_LAN_IP` (IP, отображаемый в панели TUI)
   - Порт: `27910`

4. **Глобально на Android:** Настройки → WiFi → долгое нажатие на сеть → Изменить → Дополнительно → Прокси → Вручную → Сервер: `YOUR_LAN_IP`, Порт: `27910`.
